To install the open-mediakit transcoder, you will need the following : 

- a Linux or *Nix install with a webserver and php5 (version 5.3 minimum) and php libraries (php5-curl)

- ffmpeg and imagemagick WITH proprietary codecs if you want to be able to encode to mp4, mp3 etc.

To do this, under debian Linux, use deb-multimedia repository as follow 
(add this line in /etc/apt/sources.list.d/deb-multimedia.list )

deb http://debian.octopuce.fr/debian-multimedia squeeze-backports main contrib non-free
deb http://debian.octopuce.fr/debian squeeze-backports main contrib non-free

then launch, as root : 

# aptitude update
# aptitude install -t squeeze-backports ffmpeg libavcodec-extra-53 libavdevice-extra-53 libavfilter-extra-2 libavformat-extra-53 libavutil-extra-51
(later, when we will support document parsing, we may need pdfutils, htmltotext, unoconv ...)

other recommended commands to install the Transcoder properly (on debian) : 

# aptitude install libapache2-mod-php5 apache2-mpm-prefork php5-curl php-apc mysql-server
# a2enmod rewrite

Then install a vhost pointing to your openmediakit-transcode instance (in our example with the source code in /var/www/omkt)
for example here in /etc/apache2/sites-available/omkt

<Virtualhost *:80>

  ServerName <your domain name here>

  DocumentRoot /var/www/omkt/public

  ErrorLog ${APACHE_LOG_DIR}/error.log
  CustomLog ${APACHE_LOG_DIR}/access.log combined

  <Directory /var/www/omkt>
    Order Allow,deny
    Allow from all
    AllowOverride All
  </Directory>

</VirtualHost>	  

then

# a2ensite omkt
# /etc/init.d/apache2 reload

Install the database and configure it properly in /var/www/omkt/config.php

# mysql -B -e "CREATE DATABASE omkt; GRANT ALL ON omkt.* TO 'omkt'@'localhost' IDENTIFIED BY 'randompassword';"
# cp /var/www/omkt/config.inc.php.sample /var/www/omkt/config.inc.php
# emacs /var/www/omkt/config.inc.php

Launch install.php to initialize the database (this program will also UPGRADE the database from one version to the next one.)

# /var/www/omkt/install

Finally, launch the conversion / metadata / thumbnailer processes anytime you want, either by cron, or by using some init script
